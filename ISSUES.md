## How to Report an Issue

If you encounter a bug, have a feature request, or want to suggest an improvement, please follow these steps:

### 1. **Check Existing Issues**
Before creating a new issue, please check if someone has already reported it:
- Visit: [GitHub Issues](https://github.com/StrungPattern-coder/QuestEd/issues)
- Search for keywords related to your issue

### 2. **Create a New Issue**
If your issue doesn't exist, create a new one:
- Go to: [New Issue](https://github.com/StrungPattern-coder/QuestEd/issues/new)
- Choose the appropriate template (Bug Report, Feature Request, or General Issue)

### 3. **Provide Details**
Please include as much information as possible:

## üîç Common Issues & Solutions

### "I can't log in"
- Check if you're using the correct email and password
- Try the "Forgot Password" link
- Clear your browser cache and cookies
- Try a different browser

### "Test not loading"
- Refresh the page
- Check your internet connection
- Try logging out and back in
- Clear browser cache

### "Email not received"
- Check your spam/junk folder
- Wait a few minutes (emails can be delayed)
- Verify your email address is correct
- Contact your teacher/admin

### "Live quiz not working"
- Ensure you have a stable internet connection
- Try refreshing the page
- Check if you're using the correct join code
- Verify your browser supports WebSockets

### "Features missing on mobile"
- Some features may be optimized for desktop
- Try rotating to landscape mode
- Update your browser to the latest version
- Report specific missing features as issues

## ü§ù Contributing

Want to fix an issue yourself? We welcome contributions!

### Getting Started
1. Fork the repository
2. Create a feature branch (`git checkout -b fix/issue-name`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages (`git commit -m 'Fix: resolved issue with...'`)
6. Push to your fork (`git push origin fix/issue-name`)
7. Open a Pull Request

### Code Guidelines
- Follow existing code style and conventions
- Write clear, descriptive commit messages
- Add comments for complex logic
- Test your changes thoroughly
- Update documentation if needed

### Pull Request Template
```
**Related Issue:**
Fixes #[issue-number]

**Description:**
What changes were made and why.

**Type of Change:**
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation update
- [ ] Performance improvement
- [ ] Code refactoring

**Testing:**
- [ ] Tested locally
- [ ] Added/updated tests
- [ ] All tests passing
- [ ] No console errors

**Screenshots:**
If applicable, add screenshots.

**Checklist:**
- [ ] Code follows project style
- [ ] Self-review completed
- [ ] Comments added for complex code
- [ ] Documentation updated
- [ ] No new warnings generated
```

## üéØ Issue Lifecycle

1. **Reported** - Issue is created
2. **Triaged** - Team reviews and labels
3. **Confirmed** - Issue verified and prioritized
4. **Assigned** - Someone is working on it
5. **In Progress** - Active development
6. **Testing** - Fix is being tested
7. **Resolved** - Fix merged and deployed
8. **Closed** - Issue completed
